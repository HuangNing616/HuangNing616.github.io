SparkCore学习笔记（上）

发表于 2020-05-23 更新于 2020-07-18 分类于 计算机 ， Spark ， Core 阅读次数：
本文字数： 5k 阅读时长 ≈ 5 分钟

Spark是基于内存的计算框架，它主要包括SparkCore，SparkSQL，SparkML，SparkStreaming等方面的内容，本片文章从SparkCore的角度出发，从与MR的区别，运行模式，核心RDD，代码流程，资源和任务调度，Shuffle等方面来介绍Spark

Spark与MR的区别

  1. Spark基于内存迭代计算, MR是基于磁盘迭代计算
  2. Spark中有DAG（有向无环图）
  3. MR中只有map和reduce算子（相当于Spark中的map和reduceByKey），而Spark中有很多应用不同场景的算子

Spark运行模式

  1. Local：用于本地测试，在IDEA中经常使用Local模式
  2. Standalone：Spark自带的资源调度框架，支持分布式搭建
  3. Yarn：Hadoop生态圈中的资源调度框架，Spark可以基于Yarn运行
  4. Mesos：资源调度框架

Spark核心RDD

RDD（resilient distributed dataset）也叫弹性分布式数据集

RDD的创建

接下来通过scala的伪代码来说明RDD创建的3种方式

RDD的五大特性

  1. RDD是由partiton构成
  2. 算子（函数）作用在partition上
  3. RDD之间有依赖关系
  4. 分区器作用在（K，V）格式的RDD上
  5. partition对外提供最佳计算位置，利于数据处理的本地化

RDD注意事项

  1. textFile读取HDFS文件的方法的底层原理是什么？

    该方法的底层是通过调用MR来读取HDFS文件，即要进行split，每个split对应一个block，也对应一个partition

  2. 什么是（K，V）格式的RDD？

    RDD中的数据是一个个的二元组

  3. 哪里体现了RDD的弹性（容错性）？

    • RDD的分区可多可少
    • RDD之间有依赖关系

  4. 哪里体现了RDD的分布式？

    RDD的partition是分布在多个节点（机器）上的

算子

Transformations

Transformations（转换）算子的特点是懒执行，需要执行算子（Action）触发执行

  1. map
  1. flatMap
  1. mapPartitions
  1. mapPartitionsWithIndex
  1. combineByKey

Actions

Actions（执行算子）触发Transformations执行，spark application中有一个action算子就有一个job

  1. collect
  2. count
  3. countByKey
  4. countByValue
  5. first
  6. take(num)
  7. foreach
  8. foreachPartition
  9. Reduce

Persistence

cache

  1. cache算子默认将数据存在内存中

  2. cache() = persist(StorageLevel.MEMORY_ONLY)

persist

  1. 可以手动指定持久化级别
  2. 常用的持久化级别（存储类型）：
    • MEMORY_ONLY
    • MEMORY_ONLY_SER
    • MEMORY_AND_DISK(先存到内存，内存满了在溢写磁盘)
    • MEMORY_AND_DISK_SER
  3. 存储级别的后缀”_2” 表示有副本
  4. 尽量少使用DISK_ONLY的持久化级别

checkpoint

  1. 使用场景

    当lineage（RDD之间的依赖关系构成的链） 计算非常复杂，可以尝试使用checkpoint将数据直接持久化到指定的目录

  2. 执行流程

    step1：当spark job完成之后，Spark 会从后往前回溯，找到带有checkpoint的RDD并给它做标记
    step2：回溯完成后，Spark框架会重新启动一个job,计算标记的RDD的数据，放入指定的checkpoint目录中
    step3：数据计算完成并放入目录之后，会切断RDD的依赖关系，当Spark Application执行完成之后，数据目录中的数据不会被清除

注：checkpoint要指定存储目录！！！

持久化算子之间联系和区别

  1. cache,persist,checkpoint 都是懒执行，最小持久化单位是partition
  2. cache和persist之后可以直接赋值给一个值，下次直接使用这个值（即持久化的数据），但是这个时候后面不能紧跟action算子
  3. cache和persist的数据，当application执行完成之后会自动清除，但是通过checkpoint持久化到数据目录中的数据不会被清除
